1. 产品概述
1.1 产品定位
World Cup 2026 Predictor 是一个面向中国用户的 AI 足球赛事预测平台，核心卖点是通过机器学习模型发现博彩公司赔率的定价偏差，为用户提供竞猜决策支持。产品表层是内容驱动、社交传播力强的预测平台，底层有专业的数据分析能力。
1.2 核心价值主张
赔率价值分析: 对比模型概率 vs 博彩公司赔率隐含概率，发现正期望值(+EV)的投注机会
中文本地化深度分析: 国内市场缺乏的 AI 赛前拆解报告，通俗易懂的中文解读
社交传播机制: 精美的预测卡片、公开战绩记录，红单后自然裂变传播
透明可信: 所有预测带时间戳、不可篡改，累计 ROI 公开可查
1.3 目标用户
主要用户: 有競猜/博彩需求的中国足球用户，希望通过数据分析提高竞猜胜率
次要用户: 纯足球爱好者，对世界杯赛事感兴趣，喜欢数据化视角看球
1.4 覆盖范围
核心: 2026世界杯全部赛事预测（小组赛 + 淘汰赛）
训练数据: 五大联赛（英超、西甲、德甲、意甲、法甲）+ 国家队比赛（2022世界杯至今）
市场: 中国为主，同时布局国际市场（英文版）
1.5 商业模式
以免费层吸引流量，通过订阅付费实现收入。核心增长引擎是营销和运营，而非单纯的模型准确度。红单后的社交传播是最重要的获客手段。

3. 功能需求
3.1 P0 — 产品成立的底线
没有这些功能，产品不可能上线。
3.1.1 比赛预测引擎
每场比赛输出: 胜平负概率、比分概率矩阵 Top10、大小球概率、让球盘分析
模型包含置信度指标，让用户知道模型对这场预测的确定程度
预测结果带不可篡改的时间戳，赛前发布后不可修改
3.1.2 赔率数据接入 &amp; 价值分析
接入主流博彩公司赔率数据（通过 API 或爬取）
将赔率转换为隐含概率，与模型预测概率对比
计算期望值(EV)，高亮正 EV 机会（价值信号）
按价值强度排序，帮用户快速找到最佳机会
3.1.3 公开战绩系统
每场预测赛后自动标记结果（红单/黑单）
累计战绩看板: 命中率、累计 ROI、连红记录、按盘口类型分类的表现
全部公开可查，作为信任背书页
3.1.4 社交分享卡片生成
每场预测自动生成精美的分享图片/卡片
适配微信、微博、抖音、Twitter/X 的尺寸和格式
红单后自动生成战绩截图，一键分享
分享链接带追踪参数，可知道每次分享带来多少新用户
3.1.5 世界杯赛程展示
小组赛赛程、积分榜、淘汰赛对阵图
每场比赛旁边显示预测结果和价值信号标签
3.2 P1 — 付费转化的关键
这些功能直接影响用户是否愿意付费。
3.2.1 AI 中文赛前分析报告
每场比赛赛前自动生成中文深度分析
内容包括: 两队近况对比、伤病情况、历史交锋、战术特点、模型判断依据
通俗易懂，非专业用户也能看明白
3.2.2 用户订阅体系
三层订阅: 免费 / 基础订阅 / 高阶订阅
支持支付宝、微信支付
世界杯周期套餐（赛事期间一次性付费）
3.2.3 高价值机会推送
当模型发现高 EV 机会时，主动推送给订阅用户
支持微信服务号、App 推送、邮件等渠道
3.2.4 ROI 仪表盘
实时更新的公开页面，展示模型累计投资回报率
按时间、盘口类型、置信度等维度筛选
这是最强的营销素材——用数据证明产品价值
3.2.5 世界杯模拟
蒙特卡洛模拟整届赛事，展示每队出线概率、夺冠概率
随赛事进行实时更新
3.3 P2 — 专业深度 &amp; 粘性
服务高阶用户，提升产品专业度和用户粘性。
3.3.1 数据探索面板
球队/球员维度的详细数据: xG、xA、射门、传球、防守等
近期状态趋势图、身价变化、伤病追踪
3.3.2 高级可视化
比分概率热力图
可交互的淘汰赛 bracket
伤病对概率影响的量化分析
3.3.3 多模型对比
展示不同模型（泊松、Dixon-Coles、XGBoost、集成）的预测结果并列
让用户了解模型差异，增强信任
3.3.4 用户预测 PK 排行榜
用户可提交自己的预测，与 AI 模型和其他用户 PK
排行榜增加社区属性和竞争感

5. ML 模型需求
5.1 模型目标
模型的核心目标不是“预测准确率最大化”，而是“发现赔率定价偏差”。具体评估指标: 对模型标记为正 EV 的比赛，假设每场等额下注，长期 ROI 是否为正。

6. 营销支撑功能
营销是本产品的核心增长引擎。以下功能专门为营销服务:
6.1 内容营销支持
赛前自动生成中文分析推文/短视频脚本素材，降低日常运营工作量
从开发阶段开始记录产品建设过程，作为内容素材在 B站/抖音/小红书/X 发布
6.2 红单放大机制
每场预测都有时间戳、不可篡改、公开可查——红单时才有公信力
红单后自动生成战绩截图 + “模型连红X场”标签
一键分享到各平台，带追踪参数
6.3 分享追踪
每个分享链接带 UTM 参数，可追踪来源渠道
知道哪次红单分享带来了多少注册和付费转化
7. 后台运营监控平台
运营者需要一个独立的后台看板，实时掌握产品运行状态和业务指标。
7.1 流量监控
各渠道来源（微信、微博、抖音、X、Google、直接访问）的实时和历史流量
每日 UV/PV 趋势、地域分布
每次营销动作的效果追踪（发了一条红单微博 → 带来多少注册）
7.2 用户漏斗
访问 → 注册 → 免费活跃 → 试用付费 → 正式订阅 → 续费
每一层的转化率，及时发现瓶颈环节
7.3 订阅 &amp; 收入
各层订阅人数、新增/流失趋势
MRR（月度经常性收入）、ARPU（每用户平均收入）
付费用户的留存率
7.4 内容 &amp; 预测表现
哪场比赛的预测页面浏览量最高
哪些分享卡片传播最广
红单后的流量峰值和转化数据
7.5 系统健康度
API 调用量、爬虫运行状态、模型推理延迟、错误率
建议使用 Google Analytics + PostHog/Mixpanel 等现成工具搭建，前期不需从零开发

9. 合规注意事项
中国大陆除体彩/福彩官方竞猜外，其他形式的博彩是违法的。产品在设计和营销中需要注意:
产品定位为“赛事数据分析平台”或“AI预测娱乐工具”，而非“博彩辅助工具”
产品内不直接提供下注功能或链接到博彩平台
营销话术避免“稳赚”“必赢”等诱导性用语
加入免责声明: “本平台仅提供数据分析参考，不构成任何投注建议”
